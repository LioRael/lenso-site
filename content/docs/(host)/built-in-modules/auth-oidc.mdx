---
title: auth-oidc
description: Let Runtime Console use the host's auth sessions through OIDC.
---

Install `auth-oidc` when production Runtime Console users should sign in with
the host's real auth flow instead of development bearer tokens. It depends on
`auth`; password login is the current first-party way to create the user session
that starts the OIDC flow.

```sh
lenso module install auth
lenso module install auth-password
lenso module install auth-oidc
cargo run --bin migrate
lenso serve
```

## Production config

`auth-oidc` uses module-local boot config:

```sh
APP_ENV=production
LENSO_COMPOSITION_PROFILE=core

LENSO_MODULE_AUTH_OIDC__ENABLED=true
LENSO_MODULE_AUTH_OIDC__ISSUER=https://app.example.com
LENSO_MODULE_AUTH_OIDC__CONSOLE_REDIRECT_URIS='["https://app.example.com/console/oidc/callback"]'
LENSO_MODULE_AUTH_OIDC__JWKS='{"keys":[...]}'
LENSO_MODULE_AUTH_OIDC__ID_TOKEN_PRIVATE_KEY_PEM="$OIDC_SIGNING_KEY_PEM"
```

`console_client_id` defaults to `lenso-console`. The redirect URI must match the
hosted Console callback path exactly: `/console/oidc/callback`.

Keep `id_token_private_key_pem` in your secret store. `jwks` is the public key
set matching that signing key.

## Grant Console access

The OIDC access token resolves to the same auth user id as the password session.
Grant Console scopes through `auth.console_admin_user_scopes`:

```json
{
  "usr_admin": ["console.admin", "runtime.stories.read", "auth.users.read"]
}
```

`console.admin` lets the user enter admin HTTP endpoints. Runtime story pages
also require `runtime.stories.read`; other module pages check their own
capability strings.

For the first production admin, seed the value before opening `/console`:

```sql
insert into config.setting_values (service, key, value, updated_by)
values (
  '*',
  'auth.console_admin_user_scopes',
  '{"usr_admin":["console.admin","runtime.stories.read"]}'::jsonb,
  'bootstrap'
)
on conflict (service, key) do update
set value = excluded.value,
    updated_at = now(),
    updated_by = excluded.updated_by;
```

After one admin can enter Console, edit the same key from Config.

## Browser token boundary

Do not ship a browser Console bundle with `VITE_API_AUTH_TOKEN` or a service
bearer token baked into it. In production, `/console` should rely on password
login, OIDC callback storage, and the per-user scopes above.
