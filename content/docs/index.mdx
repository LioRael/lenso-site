Lenso is an agent-ready modular app framework for Rust business systems.

The docs are split by who is doing the work:

- Host builders create and operate one deployable Lenso app.
- Module authors add business capability through manifests, routes, actions,
  runtime functions, and optional console surfaces.

## Current Public Surface

- `lenso@0.3.8` is the public Rust facade for module-authoring declarations and the `host` feature.
- `lenso-cli` scaffolds host apps with `lenso host init <dir>`.
- `@lenso/remote-module-kit@0.1.1` supports out-of-process JavaScript and TypeScript modules.
- Runtime Console is bundled into generated hosts by published CLI builds.

## What Lenso Is Not Yet

Lenso is modular-first and service-ready, but it is not a Kubernetes-first
microservice platform. Build one deployable system first, then split selected
modules when their boundaries harden.
